# 1️⃣ Project Overview

The **JARVIS AI OS Interface** is a Python-based AI control layer that allows a user to control the computer using:

• Voice commands
• Hand gestures (webcam)
• AI assistant responses
• Holographic UI overlay

The system integrates multiple modules that run together like a **mini operating system assistant**.

Architecture idea:

User → Camera / Microphone → AI Modules → System Controller → Computer Actions

# 6️⃣ Voice System Architecture

The voice assistant contains two main loops.

TTS Loop

Handles speaking responses.

Example:

JARVIS says something using pyttsx3.

---

STT Loop

Handles speech recognition.

Steps:

Microphone captures audio
Google Speech API converts audio → text
Text is analyzed for commands

---

Wake word system:

Wake word = "jarvis"

Example command:

Jarvis open youtube

System splits this into:

wake word → jarvis
command → open youtube

Then sends command to command processor.

---

# 7️⃣ Gesture System Architecture

The gesture system uses:

MediaPipe Hands

This detects **21 hand landmark points**.

Finger joints are mapped to determine gestures.

Example:

Index finger extended → move cursor

Pinch (thumb + index) → mouse click

Two fingers → scroll

Three fingers → screenshot

Closed fist → drag

Thumb up → volume up

Thumb down → volume down

# 1️⃣3️⃣ Known Issues

1. Speech recognition sometimes fails

Logs show:

Could not understand audio

Possible reasons:

Background noise
Microphone sensitivity
Fast speaking

Solution later:

Use **Vosk or Whisper STT instead of Google API**

---

2. Random mouse clicks

Gesture detection threshold is too sensitive.

Fix later:

Adjust pinch detection threshold in gesture_recognition.py.

---

3. Some commands go to AIChat instead of SystemController

Example:

Jarvis open whatsapp

AI module receives it instead of system controller.

Later we will create a **Command Router module**.
